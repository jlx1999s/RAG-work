# 简历项目描述（医疗 RAG 实习）

## 版本A：4条（按你指定的四个重点）

1. 设计并实现医疗问答的双路径检索架构（Milvus 向量检索 + LightRAG 图检索），在 LangGraph 中完成 `vector/hybrid/graph` 自适应路由，提升复杂问题的证据召回与关系推理能力。  
2. 迭代医疗文本 Chunk 策略（切块粒度、结构化元数据、重排策略），优化分块可检索性与答案引用质量，减少“检索到但答不准”的情况。  
3. 构建长短期记忆与用户画像机制：会话内短期上下文维持 + 跨轮长期偏好/事件记忆，提升多轮问答一致性、个性化与上下文延续性。  
4. 搭建评测闭环（RAGAS + 性能/稳定性指标），以 `faithfulness/context_recall/p95 latency/error rate` 等指标驱动策略迭代，支持可复现对比与回归验证。  

## 版本B：4条 STAR（对应四个重点）

1. **S/T（双路径检索）**：原系统单一路径难覆盖医疗问题中“语义检索 + 关系推理”双重需求。  
   **A**：引入向量检索与图检索双路径，并做检索模式路由。  
   **R**：复杂医疗问题的证据召回更稳定，支持不同问题形态下的检索策略匹配。  

2. **S/T（Chunk）**：医疗文档长文本与结构混杂，默认切块导致检索命中和引用质量不稳定。  
   **A**：优化切块规则与元数据（章节、来源、类型等），配合重排策略。  
   **R**：提升 chunk 可检索性和可解释性，回答的证据引用更完整。  

3. **S/T（长短期记忆用户画像）**：多轮问答中上下文断裂，用户偏好和历史信息难持续利用。  
   **A**：实现会话短期记忆 + 跨轮长期画像记忆，并在回答阶段注入记忆信息。  
   **R**：提升连续对话一致性与个性化体验，减少重复问询。  

4. **S/T（评测）**：缺少统一指标体系，难量化优化收益。  
   **A**：构建离线评测与在线评测 API，沉淀质量、性能、稳定性指标与历史记录。  
   **R**：形成“可复现、可对比、可回归”的评测闭环，支持工程化迭代。  

## 面试口播版（30秒）

我做的是医疗 RAG 助手方向的实习项目，重点做了四件事：双路径检索、Chunk 优化、长短期记忆用户画像、评测闭环。目标不是只追求模型回答流畅，而是让回答有证据、对话有连续性、优化有数据支撑。
