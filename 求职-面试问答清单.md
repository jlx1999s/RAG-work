# 面试问答清单（医疗 RAG：安全、评测、幻觉、合规）

## 四个主线速记（30秒版）

1. 双路径检索：向量检索负责语义召回，图检索负责关系推理，路由策略决定何时走哪条链路。  
2. Chunk：通过切块粒度与元数据增强提升“可检索性 + 可引用性”。  
3. 长短期记忆用户画像：短期维护会话上下文，长期沉淀用户偏好和关键事件。  
4. 评测：用质量、性能、稳定性指标衡量迭代收益，保证优化可复现。  

## 1. 安全

### Q1：你们怎么防止用户越权访问别人的会话？
**答题要点：**
- 鉴权后端以 `current_user` 为准，不信任前端传入 `user_id`
- 对 `conversation_id` 做归属校验（会话必须属于当前用户）
- API 层 + 服务层双重校验，避免单点绕过

### Q2：你们如何处理敏感医疗内容？
**答题要点：**
- 系统定位为“信息检索辅助”，不是诊断系统
- 在回答出口统一加免责声明
- 命中急症关键词（胸痛、呼吸困难、意识障碍等）触发急诊分流提醒

### Q3：如果模型输出了危险建议怎么办？
**答题要点：**
- 优先使用检索证据生成，降低“凭空建议”
- 输出阶段做医疗安全后处理（声明+分流）
- 可以扩展加规则拦截（高危词黑白名单、专家规则）

## 2. 评测

### Q4：你如何证明 RAG 方案有效？
**答题要点：**
- 使用医疗问答数据集做离线评测
- 质量指标：`faithfulness`、`answer_relevancy`、`context_recall`、`context_precision`
- 性能指标：`avg/p95 latency`、`throughput`
- 稳定性指标：`error_rate`、`empty_answer_rate`
- 对比策略：`vector_only` vs `hybrid` vs `graph_only`

### Q5：为什么不只看准确率？
**答题要点：**
- 医疗场景更关注“回答是否有依据”（faithfulness）
- 还要兼顾延迟和稳定性，才能上线
- 单一准确率无法解释系统级表现

### Q6：如何做回归验证？
**答题要点：**
- 固定评测集与参数，定期跑离线评测
- 保留评测历史，观察指标变化趋势
- 新策略上线前做 A/B 或离线对比

## 3. 幻觉与可解释性

### Q7：RAG 仍然会幻觉，你怎么控制？
**答题要点：**
- 提示词约束“仅基于给定上下文回答”
- 无证据时明确返回“未在上下文找到答案”
- 返回来源片段，支持人工核查
- 优先高质量 chunk 与重排序策略

### Q8：怎么让答案可解释？
**答题要点：**
- 输出来源文档名、chunk 片段、检索模式
- 前端展示节点 trace（路由、检索、生成）
- 对工具调用保留 audit 记录（参数、耗时、结果）

## 4. 合规与边界

### Q9：这个系统是否符合医疗合规要求？
**答题要点：**
- 当前定位是“医疗信息辅助系统”，不替代医生诊疗
- 通过声明和分流控制边界，但不等同于医疗器械合规认证
- 若商业化，需要法务/合规/医疗专家共同评审流程

### Q10：你会怎么推进到更合规的版本？
**答题要点：**
- 引入医学专家审核的数据与规则库
- 增加高风险问答人工审核流程
- 完善日志审计与数据脱敏策略
- 明确用户知情同意与隐私策略

## 5. 追问准备（高频）

### Q11：为什么做“图 + 向量”而不是纯向量？
**答题要点：**
- 向量擅长语义相似，图更适合关系推理和结构化关联
- 医疗问题常包含“实体-关系-约束”，融合更稳

### Q12：你做的最有价值工程改进是什么？
**答题要点：**
- 把“能跑”变成“可评估、可解释、可控风险”
- 重点强调安全校验、评测闭环、医疗后处理三件事

### Q13：如果让你再迭代一版，会优先做什么？
**答题要点：**
- 扩充医疗评测集并细分任务类型
- 做分层安全策略（普通问答/高危问答）
- 增加引用质量评分和冲突证据检测
